# LLM 測試工具驗證指南

## 測試步驟

1. **啟動測試前端**
   - 使用 `npm run dev:frontend` 啟動
   - 訪問 http://localhost:5173/test

2. **驗證 UI 佈局**
   - ✅ 左側欄位：控制面板和測試結果
   - ✅ 右側欄位：對話內容顯示
   - ✅ 兩欄之間有適當間距

3. **測試功能按鈕**
   - ✅ 🚀 快速測試：測試基本 LLM 功能
   - ✅ 🧪 對話測試：創建真實對話並觸發 LLM 回應
   - ✅ 🔌 API 連接測試：檢查 API 狀態
   - ✅ 🗑️ 清除訊息：清除對話內容
   - ✅ 💬 插入測試訊息：手動添加測試訊息

4. **驗證訊息顯示**
   - ✅ 最新訊息顯示在最上面（使用 reverse()）
   - ✅ 能區分用戶訊息（👤）和 AI 回應（🤖）
   - ✅ 不同類型訊息有不同的背景色和邊框
   - ✅ 顯示訊息時間戳記

5. **測試 LLM 回應**
   - 點擊「對話測試」按鈕
   - 等待幾秒鐘讓 AI 產生回應
   - 驗證右側欄位是否顯示真實的 AI 回應

6. **測試清除功能**
   - 點擊「清除訊息」按鈕
   - 驗證對話內容被清空

## 預期行為

- 啟動時顯示世界狀態
- 測試結果即時更新在左下角
- API 連接狀態顯示在左下角
- 對話內容即時更新在右側
- AI 回應會自動被標記為 🤖
- 用戶訊息會被標記為 👤

## 問題排查

如果 LLM 沒有回應：
1. 檢查 API 連接狀態是否正常
2. 確認 convex/testing.ts 中的函數正常運作
3. 查看瀏覽器控制台是否有錯誤訊息
4. 確認環境變數配置正確
